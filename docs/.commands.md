# Commands

## Development server (auto-reload)

Run this in a terminal when actively developing — it watches files and reloads on change:

```bash
.venv/bin/python -m uvicorn divvydiary_app.web:app --reload
```

Note: stop the background service first (see below) so the port is free.

## Background service (systemd --user)

The server runs as a systemd user service `divvydiary.service`, autostarted at boot
(user lingering is enabled, so it comes up before login).

Unit file: `~/.config/systemd/user/divvydiary.service`

### Start / stop / restart

```bash
systemctl --user start divvydiary
systemctl --user stop divvydiary
systemctl --user restart divvydiary
```

### Status & insights

```bash
# Current state, PID, memory, last log lines
systemctl --user status divvydiary

# Is it enabled at boot?
systemctl --user is-enabled divvydiary

# Is it currently running?
systemctl --user is-active divvydiary
```

### Logs

```bash
# Follow live logs
journalctl --user -u divvydiary -f

# Last 200 lines
journalctl --user -u divvydiary -n 200

# Logs since today
journalctl --user -u divvydiary --since today
```

### Enable / disable autostart at boot

```bash
systemctl --user enable divvydiary    # start on boot
systemctl --user disable divvydiary   # don't start on boot
```

### After editing the unit file

```bash
systemctl --user daemon-reload
systemctl --user restart divvydiary
```
